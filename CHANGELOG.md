## [v1.3.0] - 2026-07-01 10:45 (UTC+8)
### 🚀 支援 CosyVoice 2 (教育部台語資料集)
- **新增 HTTPS 反向代理**：為運行於 Port `8006` 的 CosyVoice 2 (教育部台語資料集精調) 配置 Nginx 規則，啟用 SSL 傳輸並映射至安全路徑 `https://tts.create360.ai/8006`。
- **整合至語音合成引擎**：在前端加入 `CosyVoice 2` 模型卡與參數配置面板，支援選擇預設的「溫柔女聲」與「沉穩男聲」音色。

### 🎨 UI/UX 模型分類與順序調整
- **模型分區與順序置頂**：介面重新設計，將「台語 (微帶有些許國語)」分區放在最上方置頂，「國語 (台灣特色腔調)」分區放在下方。
  - **台語** ➔ 1. CosyVoice 2 (台語模型2 / 預設選取), 2. IndexTTS (TW) (台語帶國語模型6), 3. CosyVoice 3 (台語模型4)
  - **國語** ➔ 4. IndexTTS (Main) (模型3), 5. Qwen TTS (模型1)
- **首頁狀態欄同步**：同步更新服務狀態監測欄的順序，並新增 CosyVoice 2 的線上健康度檢測（自動請求 `/voices` 確認狀態）。
- **連線狀態欄移至 Footer**：為避免外部展示時暴露後端架構，將整個狀態檢測欄從頁面頂部移至底部頁尾（Footer）。
- **全新三欄式極致佈局 (3-Column DAW Layout)**：主面板由原兩欄式改為高階專業「三欄式工作站」排版（Column 1: 文字輸入編輯區 | Column 2: 模型選擇與各參數調校區 | Column 3: 播放監聽與歷史合成紀錄區）。大幅消除因模型卡片過長而導致的大幅上下滾動，全面提升操作效率。自適應斷點設定為 `1200px`（折疊為雙欄）與 `768px`（折疊為單欄）。
- **新增麵包屑引導機制 (Breadcrumb Navigation)**：在「設定」卡片與「說明」卡片的頂部皆導入極具質感的麵包屑導航列（🏠 首頁 / 設定 或 說明），提供直覺好用的跳轉回饋。
- **商標點擊回首頁**：左上角 `333 Taiwanese TTS Hub` 品牌 Logo 正式綁定導航事件，點擊即可一鍵滑順切換回首頁主面板。
- **動態語音說明對照表 (Help Panel)**：改為由 JS 驅動的動態渲染對照表，並將「台語模型」置頂展示。相容外部展示模式切換，當「外部展示模式」開啟（ON）時，說明面板內容將自動對應顯示為安全且符合 BD 演示用之遮蔽規格描述（如：模型 6、模型 4、模型 2）。

### 🤫 外部商務展示模式 (Name Masking Mode)
- **新增「外部展示模式」切換開關**：針對 BD（商務開發）對外展示需求，在模型選擇區頂部新增了一款 iOS 質感的 Toggle 切換開關（預設為啟動 `ON`）。
- **去技術化快速範例**：徹底移除原「簡單問候」及「系統介紹」文字中對真實模型名稱（IndexTTS, CosyVoice, Qwen）的提及，代之以完全去技術化的真實商務簡報腳本（📞 業務拜訪、📰 新聞播報、🏦 客服語音、🏔 在地導覽），以防對外展示時外流機密。
- **修復載入初始化 Bug**：確保網頁首次載入時「外部展示模式」能即刻生效，無須手動重複切換。
- **動態隱藏連線狀態欄**：當「外部展示模式」啟用（ON）時，底部 Footer 的連線狀態欄會自動完全隱藏，只在展示模式關閉（OFF）時顯示，確保商務展示的安全與簡潔。
- **歷史紀錄動態遮蔽**：修正歷史合成紀錄的渲染邏輯，使其完全相容外部展示模式的動態切換。
- **新增 Footer 署名資訊**：頁尾新增維護署名與聯絡管道「維護信箱: david@aicreate360.com | david888.com」。
- **完全遮蔽真實模型名稱與技術描述**：當展示模式開啟時，所有前端區塊、卡片標題/描述、頂部狀態欄、播放器卡片與歷史生成紀錄將自動替換為別名與非技術描述，保護核心資產不外流：
  - `CosyVoice 2` ➔ **台語模型2** (描述遮蔽為：精調純台語)
  - `IndexTTS (TW)` ➔ **台語帶國語模型6** (描述遮蔽為：大量三立台劇訓練帶特色)
  - `CosyVoice 3` ➔ **台語模型4** (描述遮蔽為：大量三立戲劇訓練帶情感)
  - `IndexTTS (Main)` ➔ **模型3** (描述遮蔽為：名人或三立主播訓練)
  - `Qwen TTS` ➔ **模型1** (描述遮蔽為：台北同事音調訓練)

## [v1.2.0] - 2026-06-30 13:44 (UTC+8)
### 🎨 UI/UX 視覺與人機交互優化
- **新增雙主題切換**：整合 Catppuccin 色系，支援一鍵切換 **Frappé (深色)** 與 **Latte (淺色)**，改善對眼睛敏感使用者的視覺體驗。
- **新增懸浮固定導覽列**：頂部導覽列改為 `position: sticky;`，具備 `backdrop-filter: blur(16px)` 的懸浮毛玻璃玻璃風與投影層次。
- **使用頻率優先佈局**：將「輸入欲合成之文字」編輯區及合成按鈕調整至介面最上方，大幅減少操作視覺負擔，落實人機工學原則。
- **縮小語音合成按鈕**：將開始語音合成按鈕從 100% 滿版寬度縮小為精緻的合適寬度並置右排列。
- **播放器常駐顯示**：移除了播放器卡片預設隱藏 (`display: none;`) 的設定，使網頁在開啟時右側視覺區便能保持完整常駐，並呈現漂亮的預設狀態。

### 🔊 音頻視覺化與播放控制增強
- **新增實體進度拉條 (Progress Scrubber)**：在音源波形圖下方加入了一條具備圓形滑動鈕的水平進度條，支援拖曳快轉、點擊跳轉，並與 Canvas 上的播放垂直指引線實現精準的雙向同步。
- **新增播放指引線與滑塊**：音頻波形圖上新增了垂直播放指引線（Playhead）與發光滑塊（Scrubber Dot），讓目前播放進度具備清晰的視覺反饋。
- **新增預設音波占位符 (Mock Waveform)**：若尚未載入語音，音波區域會動態渲染由數學公式生成的灰度音軌指紋，取代原本隱形的平線，以充實介面完整度。
- **新增 CosyVoice 3 內建預設音色**：將 `jay_promptvn.wav` 從 IndexTTS 容器複製到 CosyVoice 容器的 `/asset` 中。並在前端利用 `cross_lingual` 進階推理模式，讓使用者免去手動上傳參考音檔，直接以 **周杰倫 (Jay)** 與 **溫柔女聲 (Hayley - 全媒體之花 🌸)** 當作預設音色合成！

### 🐛 Bug 修復與清理
- **修復音色選單解析錯誤**：修正 `populateIndexVoicesUI` 處理 `/audio/voices` 返回 JSON 物件時的遍歷 Bug，防止音色下拉選單留空導致 API 請求攜帶空字串觸發伺服器 `KeyError: ''`。
- **移除舊版混合內容警告**：由於 HTTPS 反向代理已架設完成，完全移除了 HTML 中不再需要的混合內容（Mixed Content）警示欄位。
